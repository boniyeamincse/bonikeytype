# Contributing to BoniKeytype ⌨️

First off, thank you for considering contributing to BoniKeytype! It's people like you that make BoniKeytype such a great tool.

## 🌈 How Can I Contribute?

### Reporting Bugs
- Use the [GitHub Issue Tracker](https://github.com/boniyeamincse/bonikeytype/issues).
- Check if the issue has already been reported.
- Use the **Bug Report** template.

### Suggesting Enhancements
- Open a [new issue](https://github.com/boniyeamincse/bonikeytype/issues/new).
- Describe the feature clearly and why it would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Install dependencies in both `client/` and `server/`.
3. If you've added code that should be tested, add tests.
4. Ensure your code follows the existing style (we use Prettier).
5. Submit a pull request!

## 🛠️ Development Setup

1. **Prerequisites**: PostgreSQL, Node.js.
2. **Backend**:
   - `cd server`
   - `npm install`
   - Copy `.env.example` to `.env` (coming soon) and set your DB.
   - `npm run dev`
3. **Frontend**:
   - `cd client`
   - `npm install`
   - `npm run dev`

## 📜 Code Style
- Use TypeScript for all new files.
- Use functional components and hooks for React.
- Use TailwindCSS for styling.

## 🤝 Community
Please follow our [Code of Conduct](CODE_OF_CONDUCT.md) in all interactions.
